====================================================================
LESSONS LEARNED FROM DEVELOPMENT CLEANUP:
====================================================================

1. ✓ Must delete appointments FIRST (they reference patient_profiles)
2. ✓ Must delete notifications BEFORE users (critical - this was missed initially!)
3. ✓ Must delete auth_sessions BEFORE users
4. ✓ Correct order prevents ALL foreign key constraint errors
5. ✓ user_roles table uses role_id (FK to roles), not a 'role' column
6. ✓ All scripts are auto-commit (no manual COMMIT needed)

====================================================================
STEP 1: BACKUP PRODUCTION DATABASE
====================================================================

⚠️ DO THIS FIRST! MANDATORY!

In pgAdmin:
1. Right-click your production database
2. Click "Backup..."
3. Choose a filename: mibo_production_backup_2026-06-25.sql
4. Save it to a safe location
5. Verify the backup file was created

====================================================================
STEP 2: VERIFY WHAT WILL BE DELETED (PRODUCTION)
====================================================================

1. Open pgAdmin
2. Connect to your PRODUCTION database (AWS RDS)
3. Open Query Tool (Tools > Query Tool)
4. Open file: PRODUCTION_VERIFY_BEFORE_CLEANUP.sql
5. Click Execute (F5)
6. Review the output carefully:

   WHAT WILL BE DELETED:
   - Patient users and profiles
   - Appointments
   - Payments
   - Blocked slots and exceptions
   - Inactive clinicians
   - Notifications
   - Auth sessions

   WHAT WILL BE KEPT:
   - Staff users (Admin, Front Desk, Managers, Care Coordinators)
   - Active clinicians
   - Availability rules
   - User roles and permissions

7. Take a screenshot or save the output for reference

====================================================================
STEP 3: RUN CLEANUP ON PRODUCTION
====================================================================

⚠️ ONLY AFTER BACKING UP AND VERIFYING!

1. In the same Query Tool window (connected to PRODUCTION)
2. Open file: PRODUCTION_CLEANUP.sql
3. READ THE WARNINGS at the top carefully
4. Take a deep breath - this is the point of no return
5. Click Execute (F5)
6. Watch the output - you should see:
   - "Deleted appointments: X"
   - "Deleted payments: X"
   - "Deleted notifications for patients: X" (critical!)
   - "Deleted patient users: X"
   - "Deleted inactive clinicians: X"
   - Final verification showing 0 counts

⚠️ If you see ANY errors:
   - STOP immediately
   - Save the error message
   - Contact support
   - DO NOT run the script again

✓ If everything completes successfully:
   - All notices appear without errors
   - Proceed to Step 4
